Problema

Clasificación multi-label de audio

  • Cada audio puede tener más de una fuente sonora.
  • El modelo entrega 80 probabilidades independientes.
  • La métrica lwlrap evalúa ranking de etiquetas correctas.
  • El ensamble ayuda cuando las ramas ordenan bien clases distintas.

Noisy fine-tune

Augmentation

Variabilidad sin cambiar etiquetas

  • Time shift para robustez temporal.
  • Time mask y frequency mask como SpecAugment liviano.
  • Gaussian noise en el tensor log-mel.
  • Pos weights calculados desde curated train.

Idea defendible

El conjunto noisy entra como fuente de cobertura y regularización, no como verdad con el mismo peso que curated.

Regularización

Control local

La validación local no decide el final

  • Los checkpoints base fueron entrenados con full curated.
  • El split local sirve como control de degradación.
  • La decisión final sale del leaderboard.
  • El score Kaggle confirma mejora real.
